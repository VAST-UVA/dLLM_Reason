# dLLM-Reason V1.0 Release Notes

**DAG-Guided Discrete Diffusion Language Models for Reasoning**

---

## Project Overview

dLLM-Reason is a research framework for enhancing reasoning capabilities of discrete diffusion language models (dLLMs) by controlling the token unmasking order via DAG (Directed Acyclic Graph) topological structures.

**Core Insight**: dLLMs generate text by iteratively unmasking tokens. By imposing a DAG structure on unmasking order вҖ” where edges encode reasoning dependencies вҖ” we can force the model to generate prerequisite reasoning steps before downstream conclusions.

**Architecture**: Three-layer decoupled design:
```
Model Layer (what to predict)  вҶҗвҶ’  Scheduler Layer (where to unmask)  вҶҗвҶ’  DAG Layer (dependency structure)
```

**Statistics**:
- 69 Python files | ~9,200 lines of code
- 31 YAML config files
- 5 test files | 2 Jupyter notebooks
- 8 scripts (5 Python + 3 Shell)

### V0.3 вҖ” DAG Core (Core Innovation)

**Goal**: Implement the novel DAG-guided unmasking mechanism.

| Component | Details |
|-----------|---------|
| `graph/dag.py` | `TokenDAG` вҖ” boolean adjacency matrix on GPU; `ready_positions()` as batch matrix op; constructors: `empty()`, `linear_chain()`, `from_edges()`, `from_levels()`; mutation: `add_edges()`, `remove_edges()`, `mutate()`; analysis: `topological_levels()`, `depth()`, `transitive_closure()` |
| `graph/templates.py` | 6 DAG templates: `chain_of_thought_dag`, `answer_first_dag`, `skeleton_then_detail_dag`, `bidirectional_dag`, `random_dag`, `interleaved_dag` |
| `graph/constraints.py` | `is_acyclic()`, `enforce_acyclicity()` (DFS back-edge removal), `topological_mutation()` |
| `scheduler/dag_scheduler.py` | `DAGScheduler` вҖ” `ready_positions()` вҶ’ intersect with masked вҶ’ sub-strategy (all_ready / confidence_topk / proportional); `AdaptiveDAGScheduler` вҖ” confidence-aware with bypass |
| `inference/dag_sampler.py` | `DAGSampler` вҖ” pre-computed schedule from topological levels, refinement steps |
| `graph/viz.py` | `draw_dag()`, `draw_unmasking_timeline()` |

**Key Design**: DAG constraints inject at scheduler layer вҖ” models need zero modification.

```python
# Core operation вҖ” one line of GPU tensor math:
ready = (~adj | is_unmasked.unsqueeze(-1)).all(dim=1)  # (B, L)
```

## Quick Start

```bash
# Install
pip install -e ".[dev,library]"

# Train MDLM
dllm-train --model mdlm --dataset gsm8k --mode pretrain

# Evaluate with DAG schedulers
dllm-eval-dags --benchmarks mbpp humaneval --dags cot skeleton

# Search optimal DAG
dllm-search --method evolutionary --budget 200

# Visualize DAGs
dllm-viz --mode templates --seq_len 32
```
